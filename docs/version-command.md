# `/sdlc:version` — Semantic Versioning & Release Management

## Overview

`/sdlc:version` manages the full semantic release workflow for any project: detects the version source, bumps the version, creates an annotated git tag, optionally generates a CHANGELOG entry, and pushes the release. It supports both file-based versioning (package.json, Cargo.toml, etc.) and tag-only projects.

Run `/sdlc:version --init` once per project to save the configuration. All subsequent releases skip auto-detection and read `.claude/version.json` directly.

---

## Setup (Run Once)

```text
/sdlc:version --init
```

Detects the project's version file, current version, and tag convention, then presents a configuration to confirm:

```text
Detected setup:
  Mode:         file
  Version file: package.json
  Version:      1.2.3
  Tag prefix:   v
  Changelog:    no (use --changelog flag to enable per release)

Suggested config:
  .claude/version.json — will be written

Does this look right? (yes / tag-only / changelog / cancel)
```

On confirmation, writes `.claude/version.json` to the project root.

---

## Basic Usage

```text
/sdlc:version patch          # 1.2.3 → 1.2.4
/sdlc:version minor          # 1.2.3 → 1.3.0
/sdlc:version major          # 1.2.3 → 2.0.0
/sdlc:version                # auto-detect from conventional commits
```

---

## Flags

| Flag | Description |
|------|-------------|
| `major` / `minor` / `patch` | Bump type (positional, optional). If omitted, the bump type is auto-detected from conventional commits since the last tag. |
| `--init` | Run the setup wizard. Writes `.claude/version.json`. Safe to re-run. |
| `--pre <label>` | Create a pre-release version. Label is typically `beta` or `rc`. Auto-increments the counter on repeated runs. See [Pre-release Workflow](#pre-release-workflow). |
| `--no-push` | Commit and tag locally, skip `git push`. |
| `--changelog` | Generate or update `CHANGELOG.md` for this release. Disabled by default; enable per-release with this flag, or set `"changelog": true` in `.claude/version.json` to default it on. |

## Pre-release Workflow

Pre-releases use the `--pre <label>` flag. The counter auto-increments:

```text
/sdlc:version minor --pre beta    # 1.2.3 → 1.3.0-beta.1
/sdlc:version --pre beta          # 1.3.0-beta.1 → 1.3.0-beta.2
/sdlc:version --pre rc            # 1.3.0-beta.2 → 1.3.0-rc.1  (label change resets counter)
/sdlc:version minor               # 1.3.0-rc.1 → 1.3.0         (graduates the pre-release)
```

Pre-releases create annotated tags (`v1.3.0-beta.1`) and update the version file (if in file mode). Changelog generation is skipped by default for pre-releases; pass `--changelog` to override.

## Changelog Generation

When `--changelog` is passed (or `"changelog": true` in config), the skill generates a [Keep a Changelog](https://keepachangelog.com) entry from commits since the last tag:

```markdown
## [1.3.0] - 2026-03-04

### Added
- User authentication with OAuth2 PKCE flow

### Fixed
- Date parsing in reports now handles timezone offsets correctly

### Changed
- Configuration format migrated from INI to YAML
```

**Conventional commit mapping:**

| Commit type | CHANGELOG section |
|-------------|-------------------|
| `feat` | Added |
| `fix` | Fixed |
| `refactor`, `perf` | Changed |
| Commits with `!` or `BREAKING CHANGE` | Noted with `(BREAKING)` in their section |
| `chore`, `docs`, `ci`, `test`, `build`, `style` | Omitted (unless user-facing) |

The skill rewrites unclear commit messages into user-facing language and merges related commits.

## Configuration: `.claude/version.json`

Generated by `--init`. Edit manually or re-run `--init` to reconfigure.

### File-based mode

```json
{
  "mode": "file",
  "versionFile": "package.json",
  "fileType": "package.json",
  "tagPrefix": "v",
  "changelog": false,
  "changelogFile": "CHANGELOG.md"
}
```

### Tag-only mode

For projects with no version file (the git tag IS the version):

```json
{
  "mode": "tag",
  "tagPrefix": "v",
  "changelog": false,
  "changelogFile": "CHANGELOG.md"
}
```

### Config fields

| Field | Type | Description |
|-------|------|-------------|
| `mode` | `"file"` \| `"tag"` | Version source. `"file"` reads/writes a version file; `"tag"` derives version from the latest git tag. |
| `versionFile` | string | Path to version file (relative to project root). Only for `mode: "file"`. |
| `fileType` | string | Version file format. Auto-detected by `--init`. See [Supported Version Files](#supported-version-files). |
| `tagPrefix` | string | Git tag prefix. Usually `"v"` (produces `v1.2.3`). Set to `""` for no prefix. |
| `changelog` | boolean | Whether to generate CHANGELOG by default. Defaults to `false`. Override per-release with `--changelog`. |
| `changelogFile` | string | Path to the CHANGELOG file. Defaults to `"CHANGELOG.md"`. |

## Supported Version Files

Auto-detected in this priority order (first match wins):

| Priority | File | Type key | Version extraction |
|----------|------|----------|--------------------|
| 1 | `package.json` | `package.json` | `.version` field |
| 2 | `Cargo.toml` | `cargo.toml` | `version = "x.y.z"` in `[package]` section |
| 3 | `pyproject.toml` | `pyproject.toml` | `version = "x.y.z"` in `[project]` or `[tool.poetry]` section |
| 4 | `pubspec.yaml` | `pubspec.yaml` | `version: x.y.z` |
| 5 | `.claude-plugin/plugin.json` | `plugin.json` | `.version` field |
| 6 | `VERSION` | `version-file` | Entire file content (trimmed) |
| 7 | `version.txt` | `version-file` | Entire file content (trimmed) |

## Requirements

- **git** — required for tagging and commit operations
- **No `gh` required** — unlike `/sdlc:pr`, this command does not need the GitHub CLI
- **`.claude/version.json`** — must exist before release runs (created by `--init`)
